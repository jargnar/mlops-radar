# An MLOps radar

This is an MLOps Radar. An ML project can be in two phases, either (A) exploration or (B) productionisation & iteration.

## Exploration phase

#### Notebooks + compute

1. [Jupyter on local machine](https://jupyter.org/install)
2. [Jupyter on K8s multi-user](https://zero-to-jupyterhub.readthedocs.io/en/latest/)
3. [Deepnote](https://deepnote.com)
4. [Google colab](https://research.google.com/colaboratory/)
5. [SageMaker Studio Lab](https://studiolab.sagemaker.aws/)
6. [GitHub Codespaces](https://github.com/features/codespaces)
7. [Databricks community](https://community.cloud.databricks.com/login.html)
8. [Kubeflow Notebooks](https://www.kubeflow.org/docs/components/notebooks/quickstart-guide/)
9. [Vertex AI Workbench](https://cloud.google.com/vertex-ai/docs/workbench)


#### Experimentation Tracking

1. [Weights & Biases](https://wandb.ai/site)
2. [MLflow](https://mlflow.org/)
3. [Tensorboard](https://www.tensorflow.org/tensorboard)
4. [Comet.ml](https://www.comet.ml/site/)
5. [Neptune.ai](https://neptune.ai/)
6. [DVC](https://dvc.org/doc/use-cases/versioning-data-and-model-files)
7. [SageMaker experiments](https://aws.amazon.com/blogs/aws/amazon-sagemaker-experiments-organize-track-and-compare-your-machine-learning-trainings/)
